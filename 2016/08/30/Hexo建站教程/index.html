发表于 2016-08-30   |  

声明：本文基于MacOS，windows类似，但可能略有差异。

写在最前面：

关于Git安装及Hexo的教程很多，官方文档也写得十分清晰，对于熟悉linux的程序员来说按照官方网站说明配置站点也就是几行命令符的事情。那我为什么还要写这篇文章呢？因为程序员和普通用户的差别不止是在于具体的代码编写上，还有解决问题的思维方式，语义的理解；这些都是刚入门的小白用户并不具备的。即使照本宣科的操作，由于每个人的系统环境总有差别，还是可能会遇到安装说明未曾提到的问题，而且就算没有遇到特殊问题，安装说明中有一句不理解就会卡住，有时想搜索问题答案都想不到合适的关键词。因此，我想写一篇让完全0基础的人都能看懂的安装指导，从而学会在最大的分享平台Github上开创属于自己的知识节点。

4. Hexo与Github

Github作为全球最火的代码仓库足够单独写一个专题，如果不清楚请移步：这里假设你已经在Github上创建了与username同名的repository即username.github.io,并且安装了Git。我们就可以通过以下步骤将Hexo网站部署到Github pages的托管服务器上。

4.1 配置SSH Key

使得本地git与远程github建立加密联系，以后更新博客不用每次都输用户名和密码

4.1.3 添加 SSH Key 公钥到 GitHub

登陆github系统，点击右上角的 Account Settings—>SSH Public keys —> add another public keys；将id_rsa.pub复制进去就可以了。

4.1.4 验证配置

出现You've successfully authenticated, but GitHub does not provide shell access.就是配置成功了。报警是因为github提供的HTTPS方式，而不是SSH方式

5. Hexo主题

Hexo主题更新是非常方便的事情，只需要把心仪的主题从git上面pull下来，然后在站点配置页里面启用一下就可以了。这里我推荐几个不错的主题。

5.1 主题推荐

  • NexT这个主题最吸引我的是分类标签，可以根据标签类型自动归类，非常的实用，可以对站点的整体知识分布一目了然。第二，其边栏的自动目录也很不错，便于长文的书写及阅读。目前本站点就是使用的该方案。
  • TKL这个主题非常漂亮，是我安装的第一套主题。最后完全是因为对标签云和边栏目录的执念让我换了NexT。
  • Maupassant看起来很美，还没试过，欢迎试过的朋友来交流。

6.2 插入图片

这里之所以把插图的方式单独提出来说，是因为markdown中的图片需要链接在线图片地址，因此直接将图片放在public－>img 文件夹下面引用，会造成正文与图片的分离，极其不方便管理，而且随着文件的增多，几乎是灾难性的。
因此这里我们使用hexo-asset-image来统一图文

  • 首先在 _config.yml 中设置 post_asset_folder:true
  • 然后在Hexo目录执行

这样在source文件目录中形成的文件结构是图片包和文件同名在一层的，发布的时候也会自动转换为相关格式
